# DSA_Basics
Covers Roadmap of learning DSA +Logic Building+Recognising Patterns in problem
Phase 1: Build Problem-Solving Logic

Before solving any problem, ask:

Step 1: Brute Force

"What is the simplest solution?"

Example:
Find largest element.

Check every element.
Complexity: O(n)

Even if it is slow, always think of brute force first.

Step 2: Optimize

Ask:

Can I avoid nested loops?
Can I use hashing?
Can I sort?
Can I use binary search?
Can I use a stack/queue?

This develops problem-solving ability.

Phase 3: Deep Learning Process

For every problem:

1. Understand
What is input?
What is output?
Constraints?
2. Think Brute Force

Write it down.

3. Find Pattern

Ask:

Array?
Hashing?
Binary Search?
Stack?
DP?
4. Write Pseudocode

Example:

for each element
    compare with current maximum
return maximum
5. Code
6. Analyze
Time Complexity?
Space Complexity?
7. Revisit

Solve again after:

1 day
7 days
30 days

This is where real learning happens.
